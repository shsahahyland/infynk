# infynk Teams Bot

Microsoft Teams bot that connects to the infynk FastAPI backend to answer engineering questions about GitHub, Confluence, Jira, and HylandDocs.

---

## 1. Prerequisites

- **Node.js 18+** — required for native `fetch` (no `node-fetch` needed)
- **ngrok account** (free tier) — [ngrok.com](https://ngrok.com) — to expose your local bot to Teams
- **Azure account** (free tier) — [portal.azure.com](https://portal.azure.com) — for Bot registration
- The **infynk backend** running locally at `http://localhost:8000`

---

## 2. Local development (Bot Framework Emulator — no Azure needed)

The fastest way to test all bot functionality before touching Azure or Teams.

```bash
# Install the Bot Framework Emulator
# Download from: https://github.com/microsoft/BotFramework-Emulator/releases

# In the teams-bot directory:
cd teams-bot
npm install

# Copy and configure the environment file
cp .env.example .env
# Leave MICROSOFT_APP_ID and MICROSOFT_APP_PASSWORD empty for local testing

# Generate placeholder manifest icons (only needed once)
node manifest/generate-icons.js

# Start the bot
node index.js
# → infynk Teams bot listening on http://localhost:3978/api/messages
```

Open Bot Framework Emulator:
1. Click **Open Bot**
2. Bot URL: `http://localhost:3978/api/messages`
3. Leave App ID and Password empty
4. Click **Connect**

Test all commands here before deploying to Teams:
- `help`
- `which team owns authentication?`
- `ingest`
- `team Payments`
- `status`

## 3. Expose to Teams with ngrok

Teams requires a public HTTPS endpoint. ngrok creates a secure tunnel to your local server.

```bash
# In a separate terminal (leave the bot running):
ngrok http 3978
```

Copy the `https://` URL from ngrok output, e.g.:
```
Forwarding  https://abc123.ngrok-free.app -> http://localhost:3978
```

Keep this terminal open — the URL changes on restart unless you have a paid ngrok account.

## 4. Azure Bot registration (free tier, step by step)

1. Go to [portal.azure.com](https://portal.azure.com) and sign in
2. Click **Create a resource** → search for **Azure Bot** → click **Create**
3. Fill in:
   - **Bot handle**: `infynk-bot` (or any unique name)
   - **Subscription**: your subscription
   - **Resource group**: create new or use existing
   - **Pricing tier**: `F0` (free — 10,000 messages/month)
   - **Microsoft App ID**: select **Create new Microsoft App ID**
4. Click **Review + create** → **Create**
5. Once deployed, go to the resource → **Configuration**
6. Set **Messaging endpoint**: `https://abc123.ngrok-free.app/api/messages`
7. Click **Save**
8. Go to **Configuration** → click **Manage Password** (next to the App ID)
9. Under **Certificates & secrets** → **New client secret** → copy the **Value**
10. Update your `.env`:
    ```
    MICROSOFT_APP_ID=<paste App ID from Azure Bot overview>
    MICROSOFT_APP_PASSWORD=<paste client secret value>
    ```
11. Restart the bot: `node index.js`

## 5. Sideload into Hyland Teams

### Create the manifest zip

```bash
cd teams-bot

# Edit manifest/manifest.json — replace both placeholder GUIDs with your Azure Bot App ID:
#   "id": "<your-azure-app-id>",
#   "bots": [{ "botId": "<your-azure-app-id>", ... }]

# Zip only the three required files:
cd manifest
zip ../infynk-teams.zip manifest.json color.png outline.png
cd ..
```

> **Note:** Replace `color.png` (192×192) and `outline.png` (32×32 transparent background, white icon) with your actual brand artwork before distributing. The placeholders generated by `generate-icons.js` work for testing.

### Upload to Teams

1. Open Microsoft Teams
2. Click **Apps** in the left sidebar
3. Click **Manage your apps** (bottom of the Apps panel)
4. Click **Upload an app** → **Upload a custom app**
5. Select `infynk-teams.zip`
6. Click **Add** to open a 1:1 chat, or **Add to a team** to add to a channel

## 6. Test these exact interactions

Once the bot is running and reachable (Emulator or Teams), verify:

| Input | Expected response |
|---|---|
| `which team owns authentication?` | Answer card with confidence bar and source citations |
| `ingest` | Ingestion result card showing documents, nodes, edges |
| `team Payments` | Confirmation message: *Active team set to Payments* |
| `status` | Status card showing Backend Online, node/edge counts |
| `help` | Help card listing all commands and active team |
| `@infynk how does BPMN task timeout work?` | Answer card (channel @mention, same as direct message) |

### Follow-up questions

After any answer, click **Ask a follow-up…** at the bottom of the card → type your follow-up in the expanded input → click **Ask**.

## Project structure

```
teams-bot/
├── index.js              Express server + CloudAdapter setup
├── bot.js                ActivityHandler — message routing, command parsing, state
├── apiClient.js          fetch() wrappers for all infynk backend endpoints
├── cards/
│   ├── answerCard.js     Answer + confidence bar + source cards Adaptive Card
│   ├── ingestCard.js     Ingestion result Adaptive Card
│   ├── statusCard.js     Health + graph stats Adaptive Card
│   └── helpCard.js       Command reference Adaptive Card
├── manifest/
│   ├── manifest.json     Teams app manifest v1.16
│   ├── generate-icons.js Generates placeholder color.png + outline.png
│   ├── color.png         192×192 app icon (generated or replaced with brand art)
│   └── outline.png       32×32 outline icon (generated or replaced with brand art)
├── package.json
├── .env.example
└── README.md
```

## Environment variables

| Variable | Default | Description |
|---|---|---|
| `MICROSOFT_APP_ID` | _(empty)_ | Azure Bot App ID — leave empty for Emulator testing |
| `MICROSOFT_APP_PASSWORD` | _(empty)_ | Azure Bot client secret — leave empty for Emulator testing |
| `PORT` | `3978` | Port the Express server listens on |
| `INFYNK_BACKEND_URL` | `http://localhost:8000` | URL of the running infynk FastAPI backend |
